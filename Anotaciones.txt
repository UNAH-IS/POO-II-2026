Función/Metodo:
    Componente que se encarga de realizar una tarea específica dentro de un programa. Las funciones pueden recibir parámetros de entrada y devolver un resultado. Son fundamentales para organizar el código y evitar la repetición.

Estandares de codificacion en Java:
    1. Nombres de clases: Deben comenzar con mayúscula y seguir el formato Upper CamelCase (ejemplo: MiClase).
    2. Nombres de métodos: Deben comenzar con minúscula y seguir el formato lower camelCase (ejemplo: miMetodo()).
    3. Nombres de variables: Deben comenzar con minúscula y seguir el formato lower camelCase (ejemplo: miVariable).
    4. Constantes: Deben escribirse en mayúsculas y usar guiones bajos para separar palabras (Snake Case)(ejemplo: MI_CONSTANTE).


Clases: Plantillas o moldes que definen las características y comportamientos de los objetos. Una clase puede contener atributos (variables) y métodos (funciones) que operan sobre esos atributos. Las clases son la base de la programación orientada a objetos (POO) y permiten crear objetos que representan entidades del mundo real.


Ejemplos clases:
    Computadora, Persona, Coche, Animal, Libro, etc. Cada una de estas clases puede tener atributos específicos (por ejemplo, una clase Persona puede tener atributos como nombre, edad y género) y métodos que definen su comportamiento (por ejemplo, una clase Coche puede tener un método para acelerar o frenar).

Importante: Una clase sería un nuevo tipo de dato, es decir, un nuevo tipo de variable. 
Por ejemplo, si creamos una clase llamada "Alumno", estaríamos definiendo un nuevo tipo de 
dato que representa a un alumno con sus atributos y métodos específicos. 
Luego, podríamos crear variables de tipo "Alumno" para almacenar información sobre 
diferentes alumnos en nuestro programa.

Modificadores de acceso: sirven para controlar la visibilidad y el acceso a los miembros de una clase (atributos y métodos).
    - default (sin modificador): El miembro es accesible solo dentro del mismo paquete.
    - public: El miembro es accesible desde cualquier clase.
    - private: El miembro es accesible solo dentro de la clase donde se declara.
    - protected: El miembro es accesible dentro de la misma clase, dentro del mismo paquete y por las subclases.


Constructores: 
    -Son métodos especiales que se utilizan para crear e inicializar objetos de una clase. El constructor tiene el mismo nombre que la clase y no tiene un tipo de retorno. 
    -Se ejecuta automáticamente cuando se crea un objeto. Si no se crea un constructor explícitamente, Java proporciona un constructor por defecto sin parámetros.
    -Se pueden crear varios contructores con diferentes parámetros (sobrecarga de constructores) para permitir diferentes formas de inicializar un objeto.

Pilares de la POO
    1. Encapsulamiento: Es el principio de ocultar los detalles internos de una clase y exponer solo lo necesario a través de métodos públicos. Esto ayuda a proteger los datos y a mantener la integridad del objeto.
    2. Abstracción: Es el proceso de simplificar la complejidad al enfocarse en los aspectos esenciales de un objeto y ocultar los detalles innecesarios. Permite crear modelos más simples y fáciles de entender.

Modificador Static:
    -El modificador static se utiliza para indicar que un miembro (atributo o método) pertenece a la clase en lugar de a una instancia específica de la clase. 
    -Los miembros estáticos son compartidos por todas las instancias de la clase, lo que significa que cualquier cambio realizado en un miembro estático afectará a todas las instancias que lo utilicen.
    -Los miembros estáticos pueden ser accedidos sin necesidad de crear un objeto de la clase, utilizando el nombre de la clase seguido del operador punto (.) y el nombre del miembro estático.

MVC 
    -Modelo-Vista-Controlador (MVC) es un patrón de diseño de software que se utiliza para separar la lógica de negocio, la presentación y el control de una aplicación. 
    -El modelo representa los datos y la lógica de negocio, la vista se encarga de la presentación y la interfaz de usuario, y el controlador maneja las interacciones del usuario y coordina la comunicación entre el modelo y la vista. 
    -Este patrón ayuda a organizar el código, mejorar la mantenibilidad y facilitar el desarrollo colaborativo.


Herencia de clases: 
    Permite crear una nueva clase (subclase o clase hija, ejemplo: Alumno, Maestro) que hereda los atributos y métodos de una clase existente (superclase o clase padre, ejemplo: Persona). La subclase puede agregar nuevos atributos y métodos, o modificar los existentes, para especializar su comportamiento. La herencia promueve la reutilización de código y la creación de jerarquías de clases.
    Automaticamente todas las clases en Java heredan de la clase Object, que es la superclase raíz de todas las clases en Java. Esto significa que todas las clases tienen acceso a los métodos definidos en la clase Object, como toString(), equals(), hashCode(), etc.
    Solo se permite herencia simple en Java, es decir, una clase solo puede extender de una única clase padre.

    Estructura de clases con herencia:

    Persona:
        nombre
        apellido
        edad
        carrera
        identidad
        genero
    
    Alumno:
        cuenta
        promedio
        clasesAprobadas

    Maestro:
        sueldo
        clasesImpartidas
        horasTrabajadas
        categoriaDocente

    Para aplicar la herencia en Java, se utiliza la palabra clave "extends". Por ejemplo:

    public class Alumno extends Persona {
        private String cuenta;
        private double promedio;
        private int clasesAprobadas;

        // Constructor, getters, setters y métodos específicos de Alumno
    }

Sobreescritura de métodos:
    Permite a una subclase proporcionar una implementación específica de un método que ya está definido en su superclase. Para sobreescribir un método, la subclase debe definir un método con el mismo nombre, tipo de retorno y parámetros que el método de la superclase. La sobreescritura se utiliza para modificar o extender el comportamiento de un método heredado.

    Ejemplo de sobreescritura de métodos:

    public class Persona {
        public void hablar() {
            System.out.println("La persona está hablando.");
        }
    }

    public class Alumno extends Persona {
        @Override
        public void hablar() {
            System.out.println("El alumno está hablando.");
        }
    }

Polimorfismo:
    Es la capacidad de un objeto de una clase hija (subclase) para ser tratado como un objeto de su clase padre (superclase). 
    Esto permite que el mismo código pueda funcionar con objetos de diferentes clases, siempre y cuando compartan una relación de herencia. El polimorfismo se logra a través de la sobreescritura de métodos y el uso de referencias de tipo superclase para referirse a objetos de tipo subclase.

    Ejemplo de polimorfismo:

    public class Persona {
        public void hablar() {
            System.out.println("La persona está hablando.");
        }
    }

    public class Alumno extends Persona {
        @Override
        public void hablar() {
            System.out.println("El alumno está hablando.");
        }
    }

    public class Maestro extends Persona {
        @Override
        public void hablar() {
            System.out.println("El maestro está hablando.");
        }
    }

    // En algún lugar del código
    Persona persona1 = new Alumno();
    Persona persona2 = new Maestro();

    persona1.hablar(); // Imprime: "El alumno está hablando."
    persona2.hablar(); // Imprime: "El maestro está hablando."

Clases abstractas:
    Son clases que no pueden ser instanciadas directamente, es decir, no se pueden crear objetos de una clase abstracta. 
    Se utilizan como base para otras clases que sí pueden ser instanciadas. Las clases abstractas pueden contener métodos abstractos (sin implementación == sin cuerpo) que deben ser implementados por las subclases, así como métodos concretos (con implementación) que pueden ser heredados por las subclases.

    A pesar de que no se pueden instanciar si pueden tener constructores, pero estos constructores solo pueden ser llamados desde las subclases a través de la palabra clave "super()".
    Una clase puede tener o no metodos abstractos, pero si una clase tiene al menos un método abstracto, entonces la clase debe ser declarada como abstracta.
    Los metodos abstractos no pueden tener un cuerpo, es decir, no pueden tener una implementación. Solo se declara la firma del método (nombre, tipo de retorno y parámetros) y se termina con un punto y coma (;), además se deben sobreescribir obligatoriamente en las clases hijas.

    Ejemplo de clase abstracta:

    public abstract class Persona {
        private String nombre;
        private String apellido;

        public Persona(String nombre, String apellido) {
            this.nombre = nombre;
            this.apellido = apellido;
        }

        public void mostrarNombreCompleto() {
            System.out.println(this.nombre + " " + this.apellido);
        }

        public abstract void guardarInformacionEnArchivo(String nombreArchivo);
    }

Interfaces:
    Son un tipo de referencia en Java similares a las clases abstractas, pero que define un conjunto de métodos abstractos (sin implementación) que una clase puede implementar. 
    Las interfaces se utilizan para definir un contrato que las clases deben cumplir.
    A diferencia de las clases abstractas, una clase puede implementar múltiples interfaces, lo que permite una forma de herencia múltiple en Java.
    Todos los metodos de una interfaz son abstractos por defecto.
    Todos los atributos de una interfaz son public static final por defecto, es decir, son constantes.
    No se puede crear objetos de una interfaz.

    Ejemplo de interfaz:

    public interface Administrable {
        public static final String NOMBRE_ARCHIVO = "informacion.txt"; // constante

        void guardarInformacionEnArchivo(String nombreArchivo);
        void cargarInformacionDesdeArchivo(String nombreArchivo);
        void actualizarInformacionEnArchivo(String nombreArchivo);
        void eliminarInformacionEnArchivo(String nombreArchivo);
    }

    Las interfaces se usan para:
    -Definir un contrato que las clases deben cumplir, lo que promueve la consistencia y la interoperabilidad entre diferentes clases.
    -Permitir la herencia múltiple.
    -Facilitar la implementación de patrones de diseño como el patrón de estrategia, el patrón de observador, etc.
    -Implementación de listeners o manejadores de eventos en aplicaciones gráficas.
